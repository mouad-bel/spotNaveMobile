Deployment

For all deployment steps, please follow the comprehensive guide available on the Flutter Delux documentation page: Flutter Delux - Deployment Guide

In this source code, you can find configuration resources within the config folder. After modifying any configuration files, don't forget to flutter pub get before you run the following commands.

App ID:

To change the application ID (package name) for your project: 

dart run package_rename -p "config/package_rename_config.yaml"

Launcher Icon:

To generate and update the launcher icons for your application: 

dart run flutter_launcher_icons -f "config/flutter_launcher_icons.yaml"

Tech Stack

Neytrip is built using a modern and efficient tech stack:

  • Routing Management: We utilize the go_router package to handle complex navigation and leverage its advanced features.
  • State Management: Primarily, we use BLoC for managing complex states. For simpler, smaller-scale states, we opt for ValueNotifier.
  • Functional Programming: For functional programming paradigms, such as implement the Either type, we integrate fpdart.
  • Local Storage: For simple data storage or session management, we rely on Shared Preferences.
  • Networking: To facilitate communication with APIs, the http package is our preferred choice for its simplicity and lightweight nature.
  • Connection Status: connectivity_plus is used to effectively monitor and discover network connectivity status.
  • Object Comparison: For simplified and efficient object comparison, we prefer using equatable.
  • Singleton / Dependency Injection: The get_it service locator is employed for robust dependency injection and managing singletons.
  • Logging: We implement logging specifically at the BLoC level to provide informative debugging without introducing complex code.
  • Spacing: gap is our go-to widget for handling empty space.
  • Network Image Handling: To handle network images such as default handle error for corrupt image or broken link, we usually use extended_image.
