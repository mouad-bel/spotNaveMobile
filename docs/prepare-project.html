Neytrip - Flutter Project Preparation Guide

Prepare Project

1. Get Project Dependencies

Flutter projects rely on various external packages and libraries defined in their pubspec.yaml file. Before running the project, you must download and make these dependencies available.

While in your project's root directory in the terminal, run the following command: 

flutter pub get

Purpose:

This command reads the pubspec.yaml file, fetches all the declared dependencies (packages) from the pub.dev repository, and makes them available for your project to compile and run. It ensures all necessary libraries are in place.

Action:

Wait for this command to complete successfully. You should see messages in your terminal indicating that packages are being downloaded, resolved, and that the process has finished.

2. Run Flutter Application

Once all dependencies are successfully fetched, you can run the Flutter application. You can debug and run your project either directly from your IDE (like VS Code or Android Studio) or via the terminal.

To run the application from the terminal, ensure you are still in your project's root directory and execute: 

flutter run

Purpose:

This command compiles your Flutter application and deploys it to a connected device or an active emulator/simulator. It then launches the app, allowing you to interact with it and see your changes in real-time (with hot reload).

For more detailed information on account setup, deployment, assets, and the project's architecture and tech stack, please refer to the comprehensive Neytrip App Guide.
